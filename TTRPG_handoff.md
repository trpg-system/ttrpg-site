# TTRPG 핸드아웃 & 타임라인 웹사이트 — 개발 인수인계 문서

## 1. 프로젝트 개요

TTRPG(테이블탑 롤플레잉 게임) 시나리오 운영을 위한 웹사이트.  
GM(게임 마스터)이 핸드아웃과 타임라인을 관리하고, 플레이어와 공유할 수 있는 툴.

---

## 2. 핵심 기능 목록

### 타임라인
- 시간/날짜 순으로 사건을 정렬해서 표시
- 사건 카드를 클릭하면 연결된 핸드아웃이 열림

### 핸드아웃
핸드아웃 하나에 담을 수 있는 내용:
- **텍스트**: 편지, 문서 등 서술형 내용
- **이미지**: 첨부 이미지
- **NPC 정보**: 이름, 설명 등
- **아이템 정보**: 이름, 설명 등

### GM 화면
- 핸드아웃 작성 / 편집 / 삭제
- 타임라인에 사건 추가 / 수정 / 삭제
- 핸드아웃 공개 여부 설정 (플레이어에게 보일지 말지)

### 플레이어 화면
- 공개된 핸드아웃만 표시
- 편집 기능 없음 (읽기 전용)

---

## 3. 사용 환경 조건

- **설치 불필요**: 브라우저에서 URL로 바로 접근 가능해야 함
- **실시간 편집**: GM이 사이트 안에서 직접 내용을 추가/수정할 수 있어야 함
- **공유 가능**: 플레이어에게 링크로 공유

→ 위 조건을 충족하려면 **백엔드 서버 + 데이터베이스**가 필요함

---

## 4. 추천 기술 스택

| 역할 | 기술 | 이유 |
|---|---|---|
| 프론트엔드 | HTML + CSS + JavaScript | 초보자 친화적, 설치 불필요 |
| 백엔드 | Node.js + Express | JavaScript와 동일한 언어로 통일 가능 |
| 데이터베이스 | SQLite 또는 JSON 파일 | 가볍고 설정이 단순함 |
| 배포 | Render / Railway (무료 플랜) | 설치 없이 URL로 접근 가능하게 해줌 |

> 복잡한 프레임워크(React 등)는 우선 사용하지 않음. 기본 웹 기술로 시작하고 필요 시 추가.

## 5. 페이지 구성

```
/ (루트)
├── /timeline       ← 타임라인 메인 화면 (GM + 플레이어 공통)
├── /gm             ← GM 전용 편집 화면
└── /player         ← 플레이어 전용 읽기 전용 화면
```

## 6. 데이터 구조 (예시)

### 사건 (Event)
```json
{
  "id": "evt_001",
  "date": "3일차 오전 10시",
  "title": "시장에서의 만남",
  "handout_id": "ho_001"
}
```

### 핸드아웃 (Handout)
```json
{
  "id": "ho_001",
  "title": "낡은 편지",
  "type": "text",
  "content": "편지 내용...",
  "image_url": "",
  "npc": "",
  "item": "",
  "is_public": true
}
```

---

## 7. 개발 우선순위 (단계별)

1. **1단계**: 타임라인 화면 + 핸드아웃 표시 (읽기만 되는 버전)
2. **2단계**: GM 편집 기능 추가 (작성/수정/삭제)
3. **3단계**: 이미지 첨부 기능
4. **4단계**: 플레이어/GM 화면 분리 + 공개 여부 설정
5. **5단계**: 배포 (외부 URL로 접근 가능하게)
